时隔几个月，Vue 项目突然"空"了——记一次 HTTP 重定向引发的 CORS 故障排查

最后更新: Jul 15, 2026 作者: Nonoxs 字数: 1.1k 预计阅读时间: 4min

背景

最近翻出一个几个月前做的 Vue 3 电商项目，想着跑起来看看效果。结果 npm run dev 之后，页面是能打开，Header、Footer 这些框架结构都正常显示，但轮播图、商品列表、分类导航——所有动态内容一片空白。

明明几个月前还好好的，怎么突然就坏了？

现象

  • 页面能正常加载，没有白屏
  • 导航栏、页头、页脚等静态布局正常渲染
  • 轮播图区域一片空白
  • 商品列表、分类菜单等数据驱动的区域全部为空
  • 浏览器 DevTools Network 面板显示 API 请求全部标红 (failed)

第四步：找到根因

关键事实：

  • 项目中 src/utils/http.js 的 baseURL 写的是 http://
  • 服务器在这几个月里升级了，对 HTTP 请求一律返回 301 重定向到 HTTPS
  • 但 301 重定向的响应中缺少 CORS 头

在浏览器中，当一个跨域请求遇到重定向时，浏览器会对整个重定向链做 CORS 校验。如果中间某个响应缺少 Access-Control-Allow-Origin，浏览器会直接阻止整个请求——这就是所有 API 调用失败的原因。

一个字母之差（http vs https），导致整个项目的数据层全部瘫痪。

总结

问题本质

环节 状态
API 服务器 升级到 HTTPS，HTTP 请求一律 301
前端代码 还在请求 HTTP 地址
301 重定向响应 缺少 CORS 头
浏览器 拦截不符合 CORS 规范的跨域重定向

这类”框架在但内容空”的排查套路

以后遇到类似情况，按这个顺序排查：

  1. 看终端 — 编译有没有报错？没报错，排除代码语法问题。
  2. 看 DevTools Network — API 请求状态是不是红色 (failed)？是的话就是请求没发出去或被拦截。
  3. 看 Console — 有没有 blocked by CORS policy 的报错？有就是跨域问题。
  4. 用 curl 测 CORS — curl -I -H "Origin: <你的域名>" <API地址>，看响应头有没有 Access-Control-Allow-Origin。
  5. 检查是否有重定向 — 如果返回 3xx，检查重定向链上每一步是否都有 CORS 头。

经验教训

  • 代码放着不动也会坏：第三方服务（尤其是教学/测试服务器）随时可能升级配置，定期跑一跑老项目是好事。
  • API 地址集中管理：不要到处硬编码，本次踩坑就是因为 Pay 组件里单独写了一份 URL，差点漏改。
  • curl 比浏览器更诚实：浏览器报错信息有时候很模糊（一个干巴巴的 net::ERR_FAILED），curl 能让你看到原始的 HTTP 响应头，省很多瞎猜的时间。
