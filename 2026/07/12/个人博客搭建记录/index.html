基于 Hexo + Klise-enhanced 主题，部署于 GitHub Pages
博客地址：https://nonoxs111.github.io/Nonoxs111-blog/

一、项目概述

项目 说明
框架 Hexo 8.x
主题 Klise-enhanced（基于 Klise 的增强版）
部署 GitHub Pages（gh-pages 分支）
CI/CD GitHub Actions 自动构建部署
仓库 github.com/Nonoxs111/Nonoxs111-blog

二、问题与解决方案

1. 404.js 报错导致 Hexo 无法启动

问题：hexo server 报错 Cannot read properties of undefined (reading 'enable')

原因：主题配置文件名 _config.klise.yml 与 _config.yml 中的主题名 Klise-enhanced 不匹配，Hexo 无法加载主题配置。

解决：将配置文件重命名为 _config.Klise-enhanced.yml。

2. 子目录部署的路径问题（核心问题）

问题：部署到 https://nonoxs111.github.io/Nonoxs111-blog/ 后出现：

  • 标签/类别/关于等页面 404
  • 头像、背景图加载失败
  • 夜间模式切换无效
  • JS/CSS 资源 404

原因：博客部署在 /Nonoxs111-blog/ 子目录下（非根目录），但模板中的静态资源使用了绝对路径（如 /img/avatar.gif、/css/main.css），浏览器请求的是 nonoxs111.github.io/img/... 而非 nonoxs111.github.io/Nonoxs111-blog/img/...。

解决：

  1. _config.yml 设置 root: /Nonoxs111-blog/
  2. 所有模板文件中的路径使用 Hexo 的 url_for() 辅助函数：
    • header.ejs：菜单链接 href="<%- url_for(url) %>"
    • index.ejs：头像 src="<%- url_for('/img/carrot.jpg') %>"
    • scripts.ejs：JS 引用 <script src="<%= url_for(url) %>">
    • head.ejs：CSS 引用已正确使用 url_for()，无需修改
  3. main.css 中背景图片使用相对路径 url('../img/fuji.jpg')

3. 背景图片过大导致加载缓慢

问题：原 fuji.png 大小为 3.3MB，首屏加载需 2-3 秒。

解决：使用 sharp 将 PNG 压缩为 JPEG，85% 品质，最终大小约 100KB（压缩了 97%）。

4. 页面切换卡顿

问题：默认的页面跳转是完整刷新，每次切换都有白屏闪烁。

解决：引入 PJAX（通过 CDN 加载 pjax@0.2.8），实现无刷新页面切换：

  • layout.ejs：添加 <div id="pjax-container"> 包裹内容区
  • scripts.ejs：初始化 PJAX，拦截菜单和内部链接
  • 移除 flying-pages.min.js（预加载插件，与 PJAX 功能重叠）

5. Canvas 雨滴动画优化

问题：雨滴效果经历多次迭代：

  • 第一版：简单泪滴形状（80 个粒子），效果不错但略显粗糙
  • 第二版：分层+渐变色（150 个粒子），导致卡顿且雨滴几乎看不见
  • 第三版：简化椭圆形状（90 个粒子），速度太慢，可见度低

最终方案：回归第一版设计——泪滴形状、80 个粒子、纯色填充、适中速度。兼顾性能与视觉。

6. 首屏布局调整（反复修改）

需求演变：

  1. 最初：常规布局，头像+文章列表在同一页
  2. 用户要求：首屏只有头像签名，文章滚动才显示
  3. 头像签名位置：用户明确要求保持在顶部，不要居中

最终方案：

  • .hero 容器：min-height: 100vh + display: flex + justify-content: space-between
  • .author（头像+名字+签名+链接）：保持在顶部，margin-top: 6.3rem
  • .scroll-hint（↓箭头）：flex 布局自动推到底部，padding-bottom: 100px
  • .posts-section（文章列表）：在 hero 下方，自然需要滚动才能看到

六、后续优化

10. 导航栏吸顶 + 滚动毛玻璃效果

需求：导航栏固定在页面顶部，向下滚动时始终可见。桌面端滚动后变为半透明毛玻璃效果，移动端保持纯白不透明。

实现：

  • 使用 position: fixed 将导航栏固定在顶部
  • 桌面端（≥769px）：默认透明，滚动超过 10px 后通过 JS 添加 .scrolled 类，触发半透明磨砂背景（background: rgba(255,255,255,0.65) + backdrop-filter: blur(12px)）
  • 移动端（≤768px）：CSS 媒体查询强制纯白背景（background: #fff），深色模式下纯黑（#131418）
  • JS 使用 requestAnimationFrame 节流，状态不变时不操作 DOM，优化性能

效果：桌面端导航栏像 iOS 那样跟着滚动毛玻璃渐变，移动端保持原本不透明样式。

11. 首页自我介绍卡片

需求：关于页添加个人介绍，风格统一，文字友好平和。

实现：

  • 在关于页顶部添加磨砂玻璃卡片，左上角浮动标签”关于我”
  • 自我介绍文字居中排版，风格自然亲切
  • 卡片与下方简历卡片样式统一（backdrop-filter: blur + background: rgba）

12. 关于页简历

需求：展示完整的简历信息，排版板正，文字可复制、链接可点。

实现：

  • 使用 HTML + CSS 内联样式编写简历卡片
  • 包含教育背景、技术栈、项目经历三个模块
  • 技术栈使用圆点列表（.tech-list），保留原始描述
  • 项目经历使用无序列表，个人博客项目采用润色后的 bullet-point 格式
  • 联系方式使用 SVG 小图标替代 emoji，更干净

13. 移动端汉堡菜单透明问题

问题：移动端点开右上角三横杠导航菜单后，背景能看到背后文字；PJAX 切换页面后透明度还会变化。

原因：汉堡菜单下拉面板的入场动画 @keyframes fadein 终点值为 opacity: 0.8，80% 不透明度的白色背景呈现半透明效果。PJAX 导航后动画状态不一致导致透明度异常。

解决：

  • 将 fadein 动画终点调整为 opacity: 0.8（保留原设计的微妙通透感）
  • 在 PJAX success 事件中添加 document.getElementById('menu-trigger').checked = false，导航后自动关闭菜单

14. PJAX 导航体验优化

问题：引入 PJAX 后出现多个体验问题。

子问题与解决：

  • 菜单高亮丢失：PJAX 不刷新导航栏，页面切换后当前菜单项不高亮。在 pjax:success 中根据 window.location.pathname 重新匹配并添加 .active 类
  • 移动端菜单未关闭：菜单链接点击跳转后汉堡菜单仍保持打开。在 pjax:success 中取消 #menu-trigger 的勾选状态
  • 代码高亮丢失：PJAX 替换内容后新页面的代码块没有 data-lang 属性。在 pjax:success 中重新遍历 figure.highlight 设置属性

15. Footer 优化

需求：底部版权声明简约化，去除冗余链接。

改动：

  • 原版：© Site by Nonoxs + Hexo 链接 + Klise 链接 + 中文网字计划链接 + “始于2025”
  • 改为：© 2025 Nonoxs. All wrongs reserved.（谐音幽默梗）
  • 移除所有外部链接，只保留一行版权信息

16. 档案馆搜索功能

需求：档案馆页面支持搜索文章。

实现：

  • 在档案馆页面顶部添加搜索输入框
  • 使用纯前端 JS 实现实时过滤：监听 input 事件 → 匹配文章标题和标签 → 隐藏不匹配的条目
  • 搜索框字体使用 font-family: inherit 与页面中文字体一致
  • 解决 PJAX 下搜索脚本不执行的问题：移除 DOMContentLoaded 包装，改用 IIFE 直接执行

17. Footer 固定在页面底部

问题：标签、类别等短内容页面，Footer 没有贴在最底部。

解决：给 #pjax-container 设置 min-height: calc(100vh - 200px)，确保内容区至少占满视口，Footer 自然下沉。

18. 动态背景与动效

需求：为页面增加视觉动效，提升浏览体验。

实现：

  • Canvas 雨滴动画：80 个蓝色泪滴形粒子，从上方下落到下方循环，带轻微水平摇摆
  • 名字淡入动画：CSS @keyframes fadeInUp，名字和签名同步从下方淡入上浮
  • 滚动箭头：首页首屏底部弹跳箭头 ↓，提示访客向下滑动查看更多内容
  • 首屏布局：使用 Flexbox 将头像签名放在首屏顶部，箭头放在底部，文章列表在首屏下方
